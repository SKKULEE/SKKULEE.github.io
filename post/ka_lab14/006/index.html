커널 아카데미 AI Lab #006 39–43일차 MLOps

2025.07.25.(FRI)

MLOps 이론

 39일차부터 42일차까지는 이인우 강사님의 실시간 MLOps 강의가 진행됐다. MLOps란, Machine Learning Operations의 약자로, 머신러닝 개발 방법론의 하나이다.
 첫 날은 데이터 직군(데이터 분석가, 데이터 엔지니어, 데이터 과학자, DB 관리자, 머신러닝 엔지니어, 비즈니스 분석가)에 대한 간략한 소개, hierarchy 속 데이터 직무간의 비교, Rule-based 모델과 ML-based 모델의 비교(Rule-based는 예측 및 설명 가능하나 구조가 경직되고 확장성이 떨어짐 / ML-based는 구조가 유연하나 내부구조 파악과 해석 및 설명이 어려움), 예시를 동반한 비즈니스 문제정의 방법, ML 모델 모니터링 요소 등에 대한 소개가 있었다.

MLOps 실습

  둘째 날부터는 AWS를 이용한 MLOps 실습이 진행됐다. 먼저, 팀장으로서 IAM 서비스를 이용해 팀원들의 IAM 계정을 생성하고 보안그룹에 추가하는 작업을 진행했다. 이후로는 EC2 서비스를 이용해 각자의 인스턴스를 생성하고 로컬에서 해당 인스턴스에 접속하는 방식으로 실습이 진행됐다.
 생성된 인스턴스 내부에서 docker를 통한 가상환경을 만들어 영화 추천 모델을 구현하고, WandB를 통해 모델의 학습 과정을 분석하고, 관련 데이터를 S3를 통해 저장하고 관리하고, docker를 통해 내부 가상 망에서 시스템 부하를 테스트해 본 후 전체 결과물을 실제 웹 사이트에 결과를 출력해보는 실습이 진행됐다.
 MLOps 강의 마지막 날인 42일차는 온라인 녹화강의 자율수강으로 이루어졌다.

후기

 이론 수업은 자세한 설명과 적절한 예시 등으로 만족스럽게 진행됐다. 실습 전반부도 잘 진행됐으나 실습 중후반부는 크게 불만족스러웠다. 먼저, 사용하는 외부 서비스(AWS, WandB, Docker, MobaXterm 등)에 대한 설명이 크게 부족했다. 이것이 정확히 어떤 서비스인지, 어떤 점이 좋은지, 다른 대안은 무엇이 있는지, 우리가 이 서비스로 지금 진행하는 작업이 정확히 무엇인지 제대로된 설명이 부족했다(이 중 일부는 금액이 청구되는 서비스다). 당연히 제대로 된 정보가 없으니 실습을 따라가는데 어려움이 있었고, 따라가는데 급급하니 이해와 학습이 제대로 이루어질 수 없었다. 이번 강의의 내용을 추후 응용할 수 있을 것이란 생각은 들지 않는다.
